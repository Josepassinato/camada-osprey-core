## Issue 1: Google Document AI Not Recognizing Service Account ✅ FIXED

### Problem
User configured `GOOGLE_APPLICATION_CREDENTIALS` with a valid service account JSON file, but the system still showed:
```
🧪 Google Document AI in MOCK MODE - No credentials provided
```

### Root Cause
The `GoogleDocumentAIProcessor` class was only checking for:
- `GOOGLE_API_KEY` (API key authentication)
- `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET` (OAuth2 authentication)

It was **NOT** checking for `GOOGLE_APPLICATION_CREDENTIALS` (service account JSON file), which is the recommended authentication method for production.

### Fix Applied

#### 1. Updated Credential Detection Logic

**File:** `backend/integrations/google/document_ai.py`

**Changes:**
```python
# Added service account credential detection
self.credentials_path = os.environ.get('GOOGLE_APPLICATION_CREDENTIALS')

# Check all three authentication methods
has_service_account = self.credentials_path and os.path.exists(self.credentials_path)
has_oauth2 = self.client_id and self.client_secret
has_api_key = self.api_key

# Not in mock mode if ANY credential method is available
self.is_mock_mode = not (has_service_account or has_oauth2 or has_api_key)
```

#### 2. Added Service Account Initialization

**New method:** `_init_service_account_credentials()`

```python
def _init_service_account_credentials(self):
    """Initialize service account credentials from JSON file"""
    try:
        if not self.credentials_path:
            logger.error("❌ Service account credentials path not set")
            return
        
        if not os.path.exists(self.credentials_path):
            logger.error(f"❌ Service account credentials file not found: {self.credentials_path}")
            return
        
        # Load service account credentials
        credentials = service_account.Credentials.from_service_account_file(
            self.credentials_path,
            scopes=['https://www.googleapis.com/auth/cloud-platform']
        )
        
        self.credentials = credentials
        logger.info("✅ Service account credentials loaded successfully")
        
        # Verify credentials work
        try:
            credentials.refresh(Request())
            logger.info("✅ Service account credentials verified")
        except Exception as e:
            logger.warning(f"⚠️ Service account credentials verification failed: {e}")
            
    except Exception as e:
        logger.error(f"❌ Service account initialization failed: {e}")
        self.credentials = None
```

#### 3. Updated Initialization Flow

**Priority order:**
1. **Service Account** (RECOMMENDED) - `GOOGLE_APPLICATION_CREDENTIALS`
2. **API Key** (LIMITED) - `GOOGLE_API_KEY`
3. **OAuth2** (USER AUTH) - `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET`

```python
if has_service_account:
    logger.info(f"🔗 Google Document AI initialized with service account for project {self.project_id}")
    logger.info(f"🔑 Credentials file: {self.credentials_path}")

    self.document_ai_endpoint = f"https://{self.location}-documentai.googleapis.com/v1/..."
    self.vision_endpoint = "https://vision.googleapis.com/v1/images:annotate"
    
    self._init_service_account_credentials()
    
elif has_api_key:
```

### Supported Authentication Methods

The system now supports all three Google Cloud authentication methods:

#### Method 1: Service Account (RECOMMENDED)
```bash
# .env configuration
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account-key.json
GOOGLE_CLOUD_PROJECT_ID=your-project-id
GOOGLE_CLOUD_LOCATION=us
GOOGLE_DOCUMENT_AI_LOCATION=us
```

**Pros:**
- ✅ Full access to all Google Cloud services
- ✅ Works with Document AI, Vision API, Speech, etc.
- ✅ Most secure for production
- ✅ No user interaction required

**Cons:**
- ⚠️ Requires Google Cloud project setup
- ⚠️ Requires service account creation
- ⚠️ Costs money (pay-as-you-go)

**Use for:** Production, staging, automated systems

---

#### Method 2: API Key (LIMITED)

**Pros:**

- ✅ Works with Vision API

**Cons:**
- ❌ Does NOT work with Document AI
- ❌ Limited functionality
- ❌ Less secure than service account

**Use for:** Development, testing Vision API only

---

#### Method 3: OAuth2 (USER AUTH)

**Pros:**
- ✅ User-based authentication
- ✅ Good for user-facing features

**Cons:**
- ⚠️ Requires user login flow
- ⚠️ Not suitable for automated systems

## Issue 2: ModuleNotFoundError - backend.maria_api ✅ FIXED

### Problem
```

File: backend/server.py
Line: import backend.maria_api
```

### Root Cause
The code had an obsolete import statement from an old file structure. The Maria API was refactored and moved to `backend/agents/maria/api.py`, but an old import remained:

```python

from backend.agents.maria.api import router as maria_api_router
```

The old import was redundant and causing the error.

### Fix Applied

**File:** `backend/server.py`

**Removed obsolete import:**

The correct import already existed on line 140, so no additional changes were needed.

## Troubleshooting

### "Service account credentials file not found"

**Check:**
```bash
# Verify file exists
ls -la $GOOGLE_APPLICATION_CREDENTIALS

# Check .env file
cat backend/.env | grep GOOGLE_APPLICATION_CREDENTIALS
```

**Fix:**
- Ensure path is absolute (not relative)
- Ensure file has correct permissions
- Ensure file is valid JSON

---

### "Service account credentials verification failed"

**Possible causes:**
1. Service account doesn't have required permissions
2. APIs not enabled in Google Cloud Console
3. Credentials expired or revoked

**Fix:**
1. Go to Google Cloud Console > IAM & Admin > Service Accounts
2. Verify service account has these roles:
   - Document AI User
   - Cloud Vision API User
   - Cloud Speech-to-Text User
3. Enable required APIs:
   - Document AI API
   - Vision API
   - Speech-to-Text API
