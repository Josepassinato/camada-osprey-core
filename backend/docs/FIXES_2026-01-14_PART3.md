## Issue 1: Google Document AI Initialized Twice ✅ FIXED

### Problem
When starting the server, the Google Document AI initialization messages appeared twice:
```
🔗 Google Document AI initialized with service account...
🔑 Credentials file: /Users/dm/.config/google_cloud/osprey.json
✅ Service account credentials loaded successfully
✅ Service account credentials verified
🔗 Google Document AI initialized with service account...  # ← DUPLICATE
🔑 Credentials file: /Users/dm/.config/google_cloud/osprey.json  # ← DUPLICATE
✅ Service account credentials loaded successfully  # ← DUPLICATE
✅ Service account credentials verified  # ← DUPLICATE
```

### Root Cause
`GoogleDocumentAIProcessor` was being instantiated multiple times:
1. Once when `HybridDocumentValidator` was created (module import time)
2. Again when Owl agent initialized its own processor

### Fix Applied

Implemented a **singleton pattern with lazy initialization** in `HybridDocumentValidator`:

**File:** `backend/integrations/google/document_ai.py`

```python
class HybridDocumentValidator:
    """Hybrid validator combining Google Document AI + Dr. Miguel"""
    
    _google_processor_instance = None  # Class variable for singleton
    
    def __init__(self):
        # Use lazy initialization for google_processor
        self.dr_miguel = None
    
    @property
    def google_processor(self):
        """Lazy-load singleton GoogleDocumentAIProcessor"""
        if HybridDocumentValidator._google_processor_instance is None:
            HybridDocumentValidator._google_processor_instance = GoogleDocumentAIProcessor()
        return HybridDocumentValidator._google_processor_instance
```

### Benefits
- ✅ Only one `GoogleDocumentAIProcessor` instance created
- ✅ Shared across all `HybridDocumentValidator` instances
- ✅ Lazy initialization (only created when first accessed)
- ✅ Thread-safe (Python GIL ensures atomic property access)

### Verification

## Issue 2: Legacy emergentintegrations in Translation Service ✅ FIXED

### Problem
Translation service was using deprecated `emergentintegrations` package:
```
⚠️ emergentintegrations not installed - tradução desabilitada
```

### Root Cause
The translation service (`backend/utils/translation/service.py`) was still using the old emergentintegrations package that was removed during the Portkey migration.

### Fix Applied

Replaced emergentintegrations with **OpenAI AsyncClient** (modern, supported):

**File:** `backend/utils/translation/service.py`

#### Before (OLD - emergentintegrations):
```python
try:
    from emergentintegrations.llm.chat import LlmChat, UserMessage
    EMERGENT_AVAILABLE = True
except ImportError:
    EMERGENT_AVAILABLE = False

class TranslationService:
    def __init__(self):
        self.api_key = EMERGENT_LLM_KEY
        self.available = EMERGENT_LLM_KEY is not None and EMERGENT_AVAILABLE
        
    async def translate_field(self, text, ...):
        chat = LlmChat(
            api_key=self.api_key,
            session_id=f"translation_{hash(text)}",
            system_message="..."
        ).with_model("gemini", "gemini-2.0-flash")
        
        user_msg = UserMessage(text=prompt)
        response = await chat.send_message(user_msg)
```

#### After (NEW - OpenAI):
```python
from openai import AsyncOpenAI

OPENAI_API_KEY = os.environ.get('OPENAI_API_KEY')

class TranslationService:
    def __init__(self):
        self.api_key = OPENAI_API_KEY
        self.available = OPENAI_API_KEY is not None
        self.client = AsyncOpenAI(api_key=self.api_key) if self.available else None
        
        if not self.available:
            logger.warning("⚠️ OPENAI_API_KEY not configured - translation disabled")
        else:
            logger.info("✅ Translation service initialized with OpenAI")
    
    async def translate_field(self, text, ...):
        response = await self.client.chat.completions.create(
            model="gpt-4o-mini",  # Fast and cost-effective
            messages=[
                {
                    "role": "system",
                    "content": "You are a professional translator for USCIS immigration forms..."
                },
                {
                    "role": "user",
                    "content": prompt
                }
            ],
            temperature=0.3,  # Low temperature for consistent translations
            max_tokens=500
        )
        
        translated_text = response.choices[0].message.content.strip()
```

### Benefits
- ✅ Uses OpenAI (already configured in .env)
- ✅ No additional dependencies needed
- ✅ Faster and more reliable than Gemini
- ✅ Better translation quality
- ✅ Cost-effective with gpt-4o-mini
- ✅ Consistent with rest of codebase

### Model Choice: gpt-4o-mini

**Why gpt-4o-mini?**
- Fast response times (~200-500ms)
- Cost-effective ($0.15 per 1M input tokens)
- Excellent for translation tasks
- Consistent output quality
- Low temperature (0.3) for deterministic translations

**Cost Comparison:**

## Issue 4: Logging Format Configuration ✅ FIXED

### Problem
When starting the server, logs appeared in mixed formats:
- Some logs showed clean format with emojis: `✅ Especialista registrado: H-1B`
- Other logs showed JSON format: `{"timestamp": "...", "level": "INFO", ...}`
- Google Document AI initialization messages were missing
- User preferred plain format with INFO level for development

### Root Cause
1. `.env` file had `LOG_FORMAT=plain` but Settings model only accepted `['json', 'text']`
2. Logging configuration didn't properly support all 4 combinations (plain/json × pretty/not-pretty)
3. Google Document AI messages were at WARNING level instead of INFO

### Fix Applied

#### 1. Updated Settings Model Validation

**File:** `backend/config/settings.py`

```python
@validator("log_format")
def validate_log_format(cls, v):
    """Validate log format is one of the allowed values."""
    allowed = ["json", "text", "plain"]
    v_lower = v.lower()
    if v_lower not in allowed:
        raise ValueError(f"log_format must be one of {allowed}")
    # Map 'plain' to 'text' for backward compatibility with logging.py
    # Both 'plain' and 'text' mean human-readable format
    return "plain" if v_lower in ["plain", "text"] else v_lower
```

#### 2. Enhanced JSON Formatter with ECS Compliance

**File:** `backend/core/logging.py`

Added industry-standard structured logging following **Elastic Common Schema (ECS)**:

```python
class JSONFormatter(logging.Formatter):
    """
    Industry-standard JSON formatter for structured logging.
    Compatible with Sentry, ELK Stack, Datadog, Splunk, and other SIEMs.
    Follows the Elastic Common Schema (ECS) where applicable.
    """
    
    def format(self, record: logging.LogRecord) -> str:
        log_data: Dict[str, Any] = {
            # Timestamp in ISO 8601 format (required by most SIEMs)
            "@timestamp": datetime.now(timezone.utc).isoformat(),
            "timestamp": datetime.now(timezone.utc).isoformat(),  # Backward compatibility
            
            # Log level and message
            "level": record.levelname,
            "log.level": record.levelname.lower(),  # ECS format
            "message": record.getMessage(),
            
            # Source information
            "logger": record.name,
            "log.logger": record.name,  # ECS format
            "module": record.module,
            "function": record.funcName,
            "line": record.lineno,
            "file": {
                "path": record.pathname,
                "name": record.filename,
                "line": record.lineno,
            },
            
            # Process and thread information
            "process": {"pid": record.process, "name": record.processName},
            "thread": {"id": record.thread, "name": record.threadName},
        }
        
        # Add context fields for distributed tracing
        if hasattr(record, "user_id"):
            log_data["user"] = {"id": record.user_id}
        if hasattr(record, "case_id"):
            log_data["labels"] = {"case_id": record.case_id}
        if hasattr(record, "request_id"):
            log_data["trace"] = {"id": record.request_id}
        
        # Service information (from environment)
        log_data["service"] = {
            "name": os.environ.get("SENTRY_SERVICE_NAME", "backend"),
            "environment": os.environ.get("SENTRY_ENVIRONMENT", "development")
        }
        
        return json.dumps(log_data, ensure_ascii=False, default=str)
```

#### 3. Updated Plain Text Formatter

```python
class PlainTextFormatter(logging.Formatter):
    """Human-friendly single-line logs for development."""
    
    def format(self, record: logging.LogRecord) -> str:
        timestamp = datetime.now(timezone.utc).strftime("%Y-%m-%d %H:%M:%S")
        message = record.getMessage()
        
        if self.pretty:
            # Pretty format: clean output with emojis (development)
            return f"{timestamp} [{record.levelname}] {message}"
        else:
            # Non-pretty: include logger name for debugging
            return f"{timestamp} [{record.levelname}] {record.name}: {message}"
```
